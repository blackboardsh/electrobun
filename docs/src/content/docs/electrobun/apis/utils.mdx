              <h2>moveToTrash</h2>

              <p>Move a file or folder on your system to the Trash (recycle bin).</p>

              <Aside type="caution">
                <p>On MacOS when you move something to trash from the finder you can open the trash can and see a "restore" button that will put the files/folders back where they were deleted from</p>
                <p>When using moveToTrash in Electrobun it moves it to the trash can but does not enable the "restore" button. To restore you will need to manually drag the files and folders back to their originating folder</p>
              </Aside>

              
```javascript
Utils.moveToTrash(absolutePath)
```

              <h2>openExternal</h2>

              <p>Open a URL in the default browser or appropriate application. Works with `http://`, `https://`, `mailto:`, custom URL schemes, and more.</p>

              <h3>Parameters</h3>

              <table>
                <thead>
                  <tr>
                    <th>Parameter</th>
                    <th>Type</th>
                    <th>Description</th>
                  </tr>
                </thead>
                <tbody>
                  <tr>
                    <td>`url`</td>
                    <td>`string`</td>
                    <td>The URL to open (e.g., "https://example.com")</td>
                  </tr>
                </tbody>
              </table>

              <h3>Returns</h3>

              <p>`boolean` - Returns `true` if the URL was opened successfully, `false` otherwise.</p>

              <h3>Examples</h3>

              
```javascript
// Open a website in the default browser
Utils.openExternal("https://example.com");

// Open an email compose window
Utils.openExternal("mailto:support@example.com?subject=Help");

// Open a custom URL scheme (if registered)
Utils.openExternal("slack://open");

// Open with file:// protocol
Utils.openExternal("file:///Users/me/Documents/report.pdf");
```


              <h2>openPath</h2>

              <p>Open a file or folder with its default application. For files, this opens them with the associated application (e.g., `.pdf` opens with the default PDF reader). For folders, this opens them in the file manager.</p>

              <h3>Parameters</h3>

              <table>
                <thead>
                  <tr>
                    <th>Parameter</th>
                    <th>Type</th>
                    <th>Description</th>
                  </tr>
                </thead>
                <tbody>
                  <tr>
                    <td>`path`</td>
                    <td>`string`</td>
                    <td>The absolute path to the file or folder</td>
                  </tr>
                </tbody>
              </table>

              <h3>Returns</h3>

              <p>`boolean` - Returns `true` if the path was opened successfully, `false` otherwise.</p>

              <h3>Examples</h3>

              
```javascript
// Open a PDF with the default PDF reader
Utils.openPath("/Users/me/Documents/report.pdf");

// Open an image with the default image viewer
Utils.openPath("/Users/me/Pictures/photo.jpg");

// Open a folder in the file manager
Utils.openPath("/Users/me/Downloads");

// Open a text file with the default text editor
Utils.openPath("/Users/me/notes.txt");
```


              <Aside type="tip">
                <p><strong>Difference between openExternal and openPath:</strong></p>
                <ul>
                  <li>`openExternal()` - Takes a URL (with protocol like `https://`, `mailto:`, `file://`)</li>
                  <li>`openPath()` - Takes a file system path (e.g., `/Users/me/file.pdf`)</li>
                </ul>
                <p>Use `openExternal()` for web URLs and email links. Use `openPath()` for local files and folders.</p>
              </Aside>

              <h2>openFileDialog</h2>

              <p>Open a file dialogue to let the user specify a file or folder and return the path to your app. Typically you would have an event handler in the browser context like clicking an "open" button, this would trigger an rpc call to bun, which would call `` openFileDialog() `` and then optionally pass the response back to the browser context via rpc after the user has made their selection</p>

              
```javascript
// To simplify this example we'll just show a dialogue after a 2 second timeout

setTimeout(async () => {

    const chosenPaths = await Utils.openFileDialog({
        startingFolder: join(homedir(), "Desktop"),
        allowedFileTypes: "*",
        // allowedFileTypes: "png,jpg",
        canChooseFiles: true,
        canChooseDirectory: false,
        allowsMultipleSelection: true,
    });

    console.log("chosen paths", chosenPaths);
 }, 2000);
```

              <h3>Return Value</h3>

              <p>Returns a `Promise<&#123; response: number &#125;>` where `response` is the 0-based index of the clicked button.</p>

              <h3>Example: Confirmation Dialog</h3>

              
```javascript
const { response } = await Utils.showMessageBox({
    type: "question",
    title: "Confirm Delete",
    message: "Are you sure you want to delete this file?",
    detail: "This action cannot be undone.",
    buttons: ["Delete", "Cancel"],
    defaultId: 1,  // Focus "Cancel" by default
    cancelId: 1    // Pressing Escape returns 1 (Cancel)
});

if (response === 0) {
    // User clicked "Delete"
    console.log("Deleting file...");
} else {
    // User clicked "Cancel" or closed the dialog
    console.log("Cancelled");
}
```


              <h3>Example: Error Dialog</h3>

              
```javascript
await Utils.showMessageBox({
    type: "error",
    title: "Error",
    message: "Failed to save file",
    detail: "The disk may be full or you may not have write permissions.",
    buttons: ["OK"]
});
```

const { response } = await Utils.showMessageBox({

              <h2>Dock Icon Visibility (macOS)</h2>

              <p>Control whether your app's icon appears in the macOS Dock. This is useful for menu bar apps, background utilities, or apps that should only show a system tray icon.</p>

              <h3>setDockIconVisible</h3>

              <p>Show or hide the app's Dock icon.</p>

              
```javascript
// Hide the dock icon (menu bar / tray-only app)
Utils.setDockIconVisible(false);

// Show the dock icon again
Utils.setDockIconVisible(true);
```


              <h3>isDockIconVisible</h3>

              <p>Check whether the app's Dock icon is currently visible.</p>

              
```javascript
if (Utils.isDockIconVisible()) {
    console.log("App is visible in Dock");
} else {
    console.log("App is hidden from Dock");
}
```


              <Aside type="tip">
                <p>When the Dock icon is hidden, the app uses `NSApplicationActivationPolicyAccessory`. When visible, it uses `NSApplicationActivationPolicyRegular`. This feature is macOS-only.</p>
              </Aside>

              <h3>Example: Tray-only App</h3>

              
```javascript
import { Utils, Tray } from "electrobun/bun";

// Hide from dock - this is a tray-only app
Utils.setDockIconVisible(false);

// Create a tray icon instead
const tray = new Tray({
    title: "My App",
    image: "views://assets/icon-template.png",
    template: true,
    width: 22,
    height: 22,
});
```


              <h2>quit</h2>

              <p>Gracefully quit the application. This fires a `before-quit` event (which can cancel the quit), then performs native cleanup (including CEF shutdown) and terminates the process.</p>

              <p>You don't need to call `quit()` directly for most quit scenarios — Electrobun automatically routes system-initiated quits (dock icon, Cmd+Q, Ctrl+C, SIGTERM, window close) through the same lifecycle. Calling `process.exit()` is also intercepted and routed through `quit()`.</p>

              
```javascript
Utils.quit()

// The quit can be cancelled via the before-quit event
Electrobun.events.on("before-quit", (e) => {
  if (hasUnsavedChanges()) {
    e.response = { allow: false };
  }
});
```


              <p>See <a href="/electrobun/apis/events#shutdown-lifecycle">Events — Shutdown Lifecycle</a> for the full list of quit triggers, shutdown sequence, and how `before-quit` compares to bun's native `process.on("exit")`.</p>

              <h2>Paths</h2>

              <p>Cross-platform access to common OS directories and app-scoped directories. All properties are synchronous getters.</p>

              
```javascript
import { Utils } from "electrobun/bun";

console.log(Utils.paths.home);      // Home directory
console.log(Utils.paths.downloads); // Downloads folder
console.log(Utils.paths.userData);  // App-specific data directory
```

              <h3>App-Scoped Directories</h3>

              <p>These paths are scoped to your application using the `identifier` and `channel` from your app's `version.json`. The values are lazy-loaded on first access.</p>

              
```javascript
Utils.paths.userData   // {appData}/{identifier}/{channel}
Utils.paths.userCache  // {cache}/{identifier}/{channel}
Utils.paths.userLogs   // {logs}/{identifier}/{channel}

// Example: app with identifier "com.mycompany.myapp", channel "canary", on macOS:
Utils.paths.userData   // ~/Library/Application Support/com.mycompany.myapp/canary
Utils.paths.userCache  // ~/Library/Caches/com.mycompany.myapp/canary
Utils.paths.userLogs   // ~/Library/Logs/com.mycompany.myapp/canary
```


              <h3>Example: Store App Data</h3>

              
```javascript
import { Utils } from "electrobun/bun";
import { join } from "path";
import { mkdirSync, writeFileSync, readFileSync } from "fs";

// Ensure the directory exists
mkdirSync(Utils.paths.userData, { recursive: true });

// Write a settings file
const settingsPath = join(Utils.paths.userData, "settings.json");
writeFileSync(settingsPath, JSON.stringify({ theme: "dark" }));

// Read it back
const settings = JSON.parse(readFileSync(settingsPath, "utf-8"));
```


              <h3>Example: Save Downloads to User's Downloads Folder</h3>

              
```javascript
import { Utils } from "electrobun/bun";
import { join } from "path";

const savePath = join(Utils.paths.downloads, "report.pdf");
await Bun.write(savePath, pdfData);
```


              <Aside type="tip">
                <p><strong>Linux XDG Support:</strong> On Linux, user directories (`documents`, `downloads`, etc.) are resolved by reading `~/.config/user-dirs.dirs`. If the file is not present, standard fallbacks like `~/Documents` are used.</p>
              </Aside>

              <h2>GlobalShortcut</h2>

              <p>Register global keyboard shortcuts that work even when your app doesn't have focus. Similar to Electron's globalShortcut module.</p>

              
```javascript
import { GlobalShortcut } from "electrobun/bun";
```


              <h3>register</h3>

              <p>Register a global keyboard shortcut with a callback function.</p>

              <table>
                <thead>
                  <tr>
                    <th>Parameter</th>
                    <th>Type</th>
                    <th>Description</th>
                  </tr>
                </thead>
                <tbody>
                  <tr>
                    <td>`accelerator`</td>
                    <td>`string`</td>
                    <td>The keyboard shortcut (e.g., "CommandOrControl+Shift+Space")</td>
                  </tr>
                  <tr>
                    <td>`callback`</td>
                    <td>`() => void`</td>
                    <td>Function to call when the shortcut is triggered</td>
                  </tr>
                </tbody>
              </table>

              <p><strong>Returns:</strong> `boolean` - true if registered successfully, false otherwise</p>

              
```javascript
const success = GlobalShortcut.register("CommandOrControl+Shift+Space", () => {
    console.log("Global shortcut triggered!");
    // Show your app, toggle a feature, etc.
});

if (!success) {
    console.log("Failed to register shortcut (may already be in use)");
}
```


              <h3>unregister</h3>

              <p>Unregister a previously registered global shortcut.</p>

              
```javascript
GlobalShortcut.unregister("CommandOrControl+Shift+Space");
```


              <h3>unregisterAll</h3>

              <p>Unregister all global shortcuts registered by your app.</p>

              
```javascript
GlobalShortcut.unregisterAll();
```


              <h3>isRegistered</h3>

              <p>Check if a shortcut is currently registered.</p>

              
```javascript
if (GlobalShortcut.isRegistered("CommandOrControl+Shift+Space")) {
    console.log("Shortcut is active");
}
```


              <h3>Accelerator Syntax</h3>

              <p>Accelerators are strings that describe keyboard shortcuts. They consist of modifier keys and a regular key, separated by `+`.</p>

              <p><strong>Modifiers:</strong></p>
              <ul>
                <li>`Command` / `Cmd` - Command key (macOS)</li>
                <li>`Control` / `Ctrl` - Control key</li>
                <li>`CommandOrControl` / `CmdOrCtrl` - Command on macOS, Control on Windows/Linux</li>
                <li>`Alt` / `Option` - Alt key (Option on macOS)</li>
                <li>`Shift` - Shift key</li>
                <li>`Super` / `Meta` / `Win` - Windows key / Super key</li>
              </ul>

              <p><strong>Keys:</strong></p>
              <ul>
                <li>Letters: `A` through `Z`</li>
                <li>Numbers: `0` through `9`</li>
                <li>Function keys: `F1` through `F12`</li>
                <li>Special: `Space`, `Enter`, `Tab`, `Escape`, `Backspace`, `Delete`</li>
                <li>Navigation: `Up`, `Down`, `Left`, `Right`, `Home`, `End`, `PageUp`, `PageDown`</li>
                <li>Symbols: `-`, `=`, `[`, `]`, `\`, `;`, `'`, `,`, `.`, `/`, `` \` ``</li>
              </ul>

              <h3>Example: Toggle App Visibility</h3>

              
```javascript
import { GlobalShortcut, BrowserWindow } from "electrobun/bun";

// Register a shortcut to show/hide the main window
GlobalShortcut.register("CommandOrControl+Shift+H", () => {
    const win = BrowserWindow.getById(1);
    if (win) {
        // Toggle visibility
        if (win.isVisible()) {
            win.hide();
        } else {
            win.show();
        }
    }
});
```


              <h3>Platform Notes</h3>

              <ul>
                <li><strong>macOS:</strong> Uses `NSEvent addGlobalMonitorForEventsMatchingMask`. Shortcuts are observed but cannot block the event from reaching other apps.</li>
                <li><strong>Windows:</strong> Uses `RegisterHotKey`. Provides exclusive access to the shortcut.</li>
                <li><strong>Linux:</strong> Uses X11 `XGrabKey`. Provides exclusive access to the shortcut. Requires X11 display server.</li>
              </ul>

              <Aside type="caution">
                <p><strong>Note:</strong> Some shortcuts may already be reserved by the operating system or other applications. If registration fails, try a different key combination.</p>
              </Aside>

              <h2>Screen</h2>

              <p>The Screen module provides information about connected displays and the cursor position. This is useful for positioning windows, detecting multi-monitor setups, and getting screen dimensions.</p>

              
```javascript
import { Screen } from "electrobun/bun";
```


              <h3>Types</h3>

              
```typescript
interface Rectangle {
  x: number;
  y: number;
  width: number;
  height: number;
}

interface Display {
  id: number;           // Unique display identifier
  bounds: Rectangle;    // Full screen bounds
  workArea: Rectangle;  // Available area (excludes dock/taskbar/menu bar)
  scaleFactor: number;  // DPI scale (e.g., 2.0 for Retina/HiDPI)
  isPrimary: boolean;   // Whether this is the primary display
}

interface Point {
  x: number;
  y: number;
}
```


              <h3>Screen.getPrimaryDisplay()</h3>

              <p>Returns the primary display information.</p>

              
```javascript
const primary = Screen.getPrimaryDisplay();
console.log(`Primary display: ${primary.bounds.width}x${primary.bounds.height}`);
console.log(`Scale factor: ${primary.scaleFactor}x`);
console.log(`Work area: ${primary.workArea.width}x${primary.workArea.height}`);
```


              <h3>Screen.getAllDisplays()</h3>

              <p>Returns an array of all connected displays.</p>

              
```javascript
const displays = Screen.getAllDisplays();
console.log(`Found ${displays.length} display(s)`);

displays.forEach((display, index) => {
  console.log(`Display ${index}: ${display.bounds.width}x${display.bounds.height}`);
  console.log(`  Position: (${display.bounds.x}, ${display.bounds.y})`);
  console.log(`  Scale: ${display.scaleFactor}x`);
  console.log(`  Primary: ${display.isPrimary}`);
});
```


              <h3>Screen.getCursorScreenPoint()</h3>

              <p>Returns the current cursor position in screen coordinates.</p>

              
```javascript
const cursor = Screen.getCursorScreenPoint();
console.log(`Cursor at: (${cursor.x}, ${cursor.y})`);
```


              <h3>Example: Center Window on Primary Display</h3>

              
```javascript
import { Screen, BrowserWindow } from "electrobun/bun";

const primary = Screen.getPrimaryDisplay();
const windowWidth = 800;
const windowHeight = 600;

// Calculate centered position
const x = Math.round((primary.workArea.width - windowWidth) / 2) + primary.workArea.x;
const y = Math.round((primary.workArea.height - windowHeight) / 2) + primary.workArea.y;

const win = new BrowserWindow({
  title: "Centered Window",
  url: "views://mainview/index.html",
  frame: {
    width: windowWidth,
    height: windowHeight,
    x: x,
    y: y,
  },
});
```


              <h3>Example: Open Window on Display with Cursor</h3>

              
```javascript
import { Screen, BrowserWindow } from "electrobun/bun";

function getDisplayAtCursor() {
  const cursor = Screen.getCursorScreenPoint();
  const displays = Screen.getAllDisplays();

  return displays.find(display => {
    const { x, y, width, height } = display.bounds;
    return cursor.x >= x && cursor.x < x + width &&
           cursor.y >= y && cursor.y < y + height;
  }) || Screen.getPrimaryDisplay();
}

const targetDisplay = getDisplayAtCursor();
console.log(`Opening window on display: ${targetDisplay.id}`);
```


              <h3>Platform Notes</h3>

              <ul>
                <li><strong>macOS:</strong> Uses `NSScreen` and `CGMainDisplayID`. Coordinates are converted from bottom-left origin to top-left origin for consistency.</li>
                <li><strong>Windows:</strong> Uses `EnumDisplayMonitors` and `GetDpiForMonitor` for scale factor.</li>
                <li><strong>Linux:</strong> Uses GDK `gdk_display_get_n_monitors` and related APIs.</li>
              </ul>

              <h2>Session</h2>

              <p>The Session module provides cookie and storage management for webview partitions. Each partition has isolated storage, allowing you to manage cookies and clear data independently.</p>

              
```javascript
import { Session } from "electrobun/bun";
```


              <h3>Types</h3>

              
```typescript
interface Cookie {
  name: string;
  value: string;
  domain?: string;
  path?: string;
  secure?: boolean;
  httpOnly?: boolean;
  sameSite?: 'no_restriction' | 'lax' | 'strict';
  expirationDate?: number; // Unix timestamp in seconds
}

interface CookieFilter {
  url?: string;
  name?: string;
  domain?: string;
  path?: string;
  secure?: boolean;
  session?: boolean;
}

type StorageType =
  | 'cookies'
  | 'localStorage'
  | 'sessionStorage'
  | 'indexedDB'
  | 'webSQL'
  | 'cache'
  | 'all';
```


              <h3>Session.fromPartition(partition)</h3>

              <p>Get or create a session for a specific partition. Partitions starting with `persist:` are stored on disk, others are ephemeral.</p>

              
```javascript
// Persistent session (survives app restarts)
const session = Session.fromPartition("persist:myapp");

// Ephemeral session (cleared when app closes)
const tempSession = Session.fromPartition("temp");
```


              <h3>Session.defaultSession</h3>

              <p>Get the default session (equivalent to `persist:default` partition).</p>

              
```javascript
const session = Session.defaultSession;
console.log(session.partition); // "persist:default"
```


              <h3>session.cookies.get(filter?)</h3>

              <p>Get cookies matching the optional filter criteria. Returns an array of Cookie objects.</p>

              
```javascript
const session = Session.fromPartition("persist:myapp");

// Get all cookies
const allCookies = session.cookies.get();
console.log(`Found ${allCookies.length} cookies`);

// Get cookies by name
const authCookies = session.cookies.get({ name: "auth_token" });

// Get cookies by domain
const domainCookies = session.cookies.get({ domain: "example.com" });

// Get cookies by URL
const urlCookies = session.cookies.get({ url: "https://api.example.com" });
```


              <h3>session.cookies.set(cookie)</h3>

              <p>Set a cookie. Returns `true` if successful.</p>

              
```javascript
const session = Session.fromPartition("persist:myapp");

// Set a basic cookie
session.cookies.set({
  name: "user_id",
  value: "12345",
  domain: "myapp.com",
  path: "/"
});

// Set a secure cookie with expiration
session.cookies.set({
  name: "auth_token",
  value: "abc123xyz",
  domain: "api.myapp.com",
  path: "/",
  secure: true,
  httpOnly: true,
  sameSite: "strict",
  expirationDate: Math.floor(Date.now() / 1000) + (7 * 24 * 60 * 60) // 7 days
});
```


              <h3>session.cookies.remove(url, name)</h3>

              <p>Remove a specific cookie by URL and name. Returns `true` if successful.</p>

              
```javascript
const session = Session.fromPartition("persist:myapp");

// Remove a specific cookie
session.cookies.remove("https://myapp.com", "user_id");
```


              <h3>session.cookies.clear()</h3>

              <p>Clear all cookies for this session.</p>

              
```javascript
const session = Session.fromPartition("persist:myapp");

// Clear all cookies
session.cookies.clear();
```


              <h3>session.clearStorageData(types?)</h3>

              <p>Clear storage data for this session. You can specify which types to clear, or use `'all'` to clear everything.</p>

              
```javascript
const session = Session.fromPartition("persist:myapp");

// Clear all storage
session.clearStorageData();

// Clear specific storage types
session.clearStorageData(['cookies', 'localStorage']);

// Clear just the cache
session.clearStorageData(['cache']);
```


              <h3>Example: User Logout</h3>

              
```javascript
import { Session } from "electrobun/bun";

function logout() {
  const session = Session.fromPartition("persist:user");

  // Clear auth cookies
  session.cookies.remove("https://api.myapp.com", "auth_token");
  session.cookies.remove("https://api.myapp.com", "refresh_token");

  // Clear local storage data
  session.clearStorageData(['localStorage', 'sessionStorage']);

  console.log("User logged out");
}
```


              <h3>Example: Multi-account Support</h3>

              
```javascript
import { Session, BrowserWindow } from "electrobun/bun";

// Create a webview with its own session
function createAccountWindow(accountId: string) {
  const partition = `persist:account-${accountId}`;
  const session = Session.fromPartition(partition);

  // Each account has isolated cookies/storage
  const win = new BrowserWindow({
    title: `Account: ${accountId}`,
    url: "https://myapp.com/dashboard",
    partition: partition,
    frame: { width: 800, height: 600, x: 100, y: 100 }
  });

  return { window: win, session };
}
```


              <h3>Platform Notes</h3>

              <ul>
                <li><strong>macOS:</strong> Uses `WKHTTPCookieStore` for WebKit and CEF's cookie manager.</li>
                <li><strong>Windows:</strong> Uses `ICoreWebView2CookieManager` for WebView2.</li>
                <li><strong>Linux:</strong> Uses `WebKitCookieManager` for WebKit2GTK and CEF's cookie manager.</li>
              </ul>

              <Aside type="tip">
                <p><strong>Partition Naming:</strong> Use `persist:` prefix for persistent storage (e.g., `persist:myapp`). Sessions without this prefix are ephemeral and cleared when the app closes.</p>
              </Aside>
