title: "CLI Commands"
---
              
              <p>The Electrobun CLI provides commands for initializing new projects and building your applications for different environments.</p>

## Installation

              <p>When you install Electrobun, the CLI tool is added to your `node_modules/bin` folder:</p>

              
```
bun install electrobun
```


              <p>This makes the `electrobun` command available in your npm scripts or via `bunx`/`npx`.</p>

## Commands

### `electrobun init`

              <p>Initializes a new Electrobun project with starter templates.</p>

#### Usage

              
```bash
# Interactive template selection
electrobun init

# Direct template selection
electrobun init [template-name]
```


#### Available Templates

              <ul>
                <li>`hello-world` - Basic single-window application</li>
                <li>`photo-booth` - Camera app with photo capture functionality</li>
                <li>`interactive-playground` - An interactive playground of Electrobun apis</li>
                <li>`multitab-browser` - Multi-tabbed web browser</li>
              </ul>

#### Examples

              
```bash
# Choose template interactively
bunx electrobun init

# Initialize with photo-booth template directly
bunx electrobun init photo-booth

# Initialize with multitab-browser template
bunx electrobun init multitab-browser
```


### `electrobun build`

              <p>Builds your Electrobun application according to the configuration in `electrobun.config.ts`.</p>

#### Usage

              
```
electrobun build [options]
```


#### Options

              <table class="options-table">
                <thead>
                  <tr>
                    <th>Option</th>
                    <th>Description</th>
                    <th>Values</th>
                    <th>Default</th>
                  </tr>
                </thead>
                <tbody>
                  <tr>
                    <td><code style="text-wrap:nowrap;">--env</code></td>
                    <td>Build environment</td>
                    <td>`dev`, `canary`, `stable`</td>
                    <td>`dev`</td>
                  </tr>
                </tbody>
              </table>

              <p>Builds always target the current host platform and architecture. To build for multiple platforms, use CI runners for each OS/architecture (see <a href="/electrobun/guides/cross-platform-development">Cross-Platform Development</a>).</p>

#### Examples

              
```bash
# Development build for current platform
electrobun build

# Development build with environment flag
electrobun build --env=dev

# Canary build
electrobun build --env=canary

# Stable (production) build
electrobun build --env=stable
```


### `electrobun run`

              <p>Launches an already-built dev bundle. This is useful when you've already run `electrobun build` and just want to relaunch the app without rebuilding.</p>

#### Usage

              
```
electrobun run
```


### `electrobun dev`

              <p>Builds your application in dev mode and then launches it. This is the primary command for day-to-day development — equivalent to running `electrobun build --env=dev` followed by `electrobun run`.</p>

#### Usage

              
```
electrobun dev [options]
```


#### Options

              <table class="options-table">
                <thead>
                  <tr>
                    <th>Option</th>
                    <th>Description</th>
                  </tr>
                </thead>
                <tbody>
                  <tr>
                    <td><code style="text-wrap:nowrap;">--watch</code></td>
                    <td>Watch source files for changes and automatically rebuild + relaunch</td>
                  </tr>
                </tbody>
              </table>

#### Examples

              
```bash
# Build and launch in dev mode
electrobun dev

# Build, launch, and watch for changes
electrobun dev --watch
```


#### Watch Mode

              <p>When using `--watch`, Electrobun automatically watches your source directories for file changes:</p>

              <ul>
                <li>The directory containing your `build.bun.entrypoint`</li>
                <li>Directories containing each view entrypoint</li>
                <li>Source paths from `build.copy`</li>
                <li>Any additional paths specified in `build.watch`</li>
              </ul>

              <p>When a file change is detected, the running app is killed, a fresh build runs (including all lifecycle hooks like `postBuild`), and the app is relaunched. File watchers are paused during builds to avoid false triggers from build output.</p>

              <p>Changes are debounced (300ms) so rapid saves only trigger a single rebuild. If a build fails, the error is logged and the watcher keeps running — the app will rebuild on the next file change.</p>

              <p>Use `build.watchIgnore` in your config to exclude files from triggering rebuilds (e.g., generated assets). See <a href="/electrobun/apis/cli/build-configuration">Build Configuration</a> for details.</p>

              <p>Press <kbd>Ctrl+C</kbd> to stop the app and exit watch mode.</p>

## Build Environments

### Development (`dev`)

              <ul>
                <li>Outputs logs and errors to terminal</li>
                <li>No code signing or notarization</li>
                <li>Creates build in `build/` folder</li>
                <li>No artifacts generated</li>
                <li>Fast iteration for testing</li>
              </ul>

### Canary

              <ul>
                <li>Pre-release/beta builds</li>
                <li>Optional code signing and notarization</li>
                <li>Generates distribution artifacts</li>
                <li>Creates update manifests for auto-updates</li>
                <li>Suitable for testing with limited users</li>
              </ul>

### Stable

              <ul>
                <li>Production-ready builds</li>
                <li>Full code signing and notarization (if configured)</li>
                <li>Optimized and compressed artifacts</li>
                <li>Ready for distribution to end users</li>
                <li>Generates all update files</li>
              </ul>

## Build Script Examples

### Basic Setup

              
```javascript
// package.json
{
  "scripts": {
    "start": "electrobun run",
    "dev": "electrobun dev",
    "dev:watch": "electrobun dev --watch",
    "build:canary": "electrobun build --env=canary",
    "build:stable": "electrobun build --env=stable"
  }
}
```


### CI Build Scripts

              <p>For multi-platform distribution, run the same build command on each platform's CI runner:</p>

              
```javascript
// package.json
{
  "scripts": {
    "build:dev": "electrobun build",
    "build:canary": "electrobun build --env=canary",
    "build:stable": "electrobun build --env=stable"
  }
}
```
