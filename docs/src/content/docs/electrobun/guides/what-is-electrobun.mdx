---
title: "What is Electrobun?"
---
  
  <p>Electrobun is a desktop application framework that lets you build ultra fast, tiny, and cross-platform applications using TypeScript. It combines the best of native performance with web development simplicity.</p>
  <h2>The Problem</h2>
  
  <p>Traditional desktop app frameworks force you to choose between developer experience and app performance:</p>
  
  <ul>
    <li><strong>Electron:</strong> Great DX but huge bundle sizes (150MB+), slow startup times (2-5s), and massive update downloads</li>
    <li><strong>Native development:</strong> Great performance but complex setup, platform-specific code, and limited web technology integration</li>
    <li><strong>Tauri:</strong> Better than Electron but still large updates and you have to learn Rust</li>
  </ul>
  <h2>The Solution</h2>
  <p>Electrobun provides a third option that doesn't compromise:</p>
  <ul>
    <li><strong>Ultra-small bundles:</strong> ~14MB compressed (90%+ smaller than Electron)</li>
    <li><strong>Lightning-fast startup:</strong> &lt;50ms cold start vs 2-5s for Electron</li>
    <li><strong>Tiny updates:</strong> 14KB patches using custom binary diff vs 100MB+ with Electron</li>
    <li><strong>Pure TypeScript:</strong> Write TypeScript for both main process and UI</li>
    <li><strong>Web technologies:</strong> HTML, CSS, JavaScript - use any frontend framework</li>
    <li><strong>Native performance:</strong> Zig bindings with Bun runtime for maximum speed</li>
    <li><strong>Optional CEF:</strong> bundle CEF (Chromium) when cross-platform consistency matters most.</li>
  </ul>

  <h2>Technical Architecture</h2>
  <p>Electrobun achieves its performance through a carefully designed architecture:</p>
  <h3>Zig, and Native Bindings</h3>
  <p>Native functionality like window management, system trays, and app menus writtin in C++ and Objc</p>
  <h3>Bun Runtime</h3>
  <p>The main process runs on Bun, providing lightning-fast Typescript execution and built-in bundling without the overhead of Node.js and V8.</p>
  <h3>System WebView</h3>
  <p>Instead of distributing Chromium, By default Electrobun uses your system's native WebView (WebKit on macOS, Edge WebView2 on Windows, WebKitGTK on Linux).</p>
  <h3>Custom Update System</h3>
  <p>Binary diff updates using a SIMD optimized BSDIFF implementation written in zig to allow for incredibly small update patches - often just kilobytes instead of megabytes.</p>
  <h3>ZSTD self-extracting distributables</h3>
  <p>The Electrobun cli bundles your app, then compresses it with state of the art compression making initial downloads as small as possible.</p>
  <h3>Custom OOPIF Implementation</h3>
  <p>Use OOPIFs (super iframes) in your html for secure, isolated, webviews across browser engines and platforms.</p>
  <h2>Key Benefits</h2>
  <h3>🚀 Faster Development</h3>
  <ul>
    <li>Fast build times - Electrobun cli uses pre-built binaries for your target platform</li>
    <li>Use any web framework (React, SolidJS, Vue, Svelte, etc.)</li>
    <li>TypeScript throughout - no context switching</li>
    <li>Built-in bundling and optimization</li>
  </ul>
  <h3>📦 Better Distribution</h3>
  <ul>
    <li>14MB bundles vs 150MB+ with Electron</li>
    <li>Kilobyte updates instead of megabyte downloads</li>
    <li>Built-in code signing and notarization</li>
    <li>Cross-platform builds from any OS</li>
    <li>Built-in ZSTD self-extractor</li>
  </ul>
  <h3>⚡ Superior Performance</h3>
  <ul>
    <li>Sub-50ms startup times</li>
    <li>Minimal memory footprint</li>
    <li>Native-feeling UI responsiveness</li>
    <li>Battery-efficient operation</li>
  </ul>
  <h3>🔐 Security First</h3>
  <ul>
    <li>Process isolation by default</li>
    <li>Secure, encrypted, and typed RPC between processes</li>
    <li>custom views:// schema for loading bundled assets in webviews</li>
    <li>Minimal attack surface</li>
  </ul>
  <h2>When to Use Electrobun</h2>
  <p>Electrobun is perfect for:</p>
  <ul>
    <li><strong>Startup MVPs:</strong> Ship fast, iterate quickly with small updates</li>
    <li><strong>Developer tools:</strong> IDEs, terminals, productivity apps that need native performance</li>
    <li><strong>Cross-platform apps:</strong> One codebase, native feel everywhere</li>
    <li><strong>High-performance apps:</strong> When Electron is too slow but native development is too complex</li>
    <li><strong>Bandwidth-conscious apps:</strong> Frequent updates without user friction</li>
    <li><strong>Multi-tab web browsers</strong>Build multi-tab experiences and mix CEF and Webkit webviews</li>
  </ul>
