---
title: "Migrating from 0.x to v1"
---
  <p>Electrobun v1 includes breaking changes to artifact naming and configuration. This guide covers what changed and how to migrate your existing apps.</p>

  <h2>Breaking Changes</h2>

  <h3>1. Config: `bucketUrl` renamed to `baseUrl`</h3>

  <p>The `release.bucketUrl` configuration option has been renamed to `release.baseUrl` to better reflect that it supports any HTTP host, not just S3-style buckets.</p>

  <h4>Before:</h4>
  
```javascript
// electrobun.config.ts
export default {
  // ...
  release: {
    bucketUrl: "https://example.com/releases",
  },
};
```


  <h4>After:</h4>
  
```javascript
// electrobun.config.ts
export default {
  // ...
  release: {
    baseUrl: "https://example.com/releases",
  },
};
```


  <h3>2. Artifact Naming: Folder-based to Prefix-based (Flat Structure)</h3>

  <p>Artifact URLs have changed from a folder-based structure to a flat prefix-based structure. This enables hosting on GitHub Releases and other platforms that don't support folders.</p>

  <h4>Before (folder-based):</h4>
  
```
https://example.com/releases/canary-macos-arm64/update.json
https://example.com/releases/canary-macos-arm64/MyApp.app.tar.zst
https://example.com/releases/canary-macos-arm64/abc123.patch
```


  <h4>After (prefix-based / flat):</h4>
  
```
https://example.com/releases/canary-macos-arm64-update.json
https://example.com/releases/canary-macos-arm64-MyApp.app.tar.zst
https://example.com/releases/canary-macos-arm64-abc123.patch
```


  <p>The local `artifacts/` folder is now flat with prefixed filenames instead of containing platform subfolders.</p>

  <h2>Migration Steps</h2>

  <p>If you have an app already deployed with the old folder-based structure, follow these steps to migrate your users:</p>

  <h3>Step 1: Build a Transitional Release</h3>

  <p>Before upgrading to v1, build and release one final version using your current Electrobun version (pre-v1). This ensures all your users update to a version that can still read the old URL structure.</p>

  
```
# Using your current Electrobun version (pre-v1)
bun run build --env=canary  # or stable
# Deploy artifacts to your bucket with the OLD folder structure
```


  <h3>Step 2: Upgrade Electrobun</h3>

  <p>Update your Electrobun dependency to v1:</p>

  
```
bun add electrobun@latest
```


  <h3>Step 3: Update Your Config</h3>

  <p>Rename `bucketUrl` to `baseUrl` in your `electrobun.config.ts`:</p>

  
```javascript
release: {
  baseUrl: "https://example.com/releases",  // was: bucketUrl
},
```


  <h3>Step 4: Switch Your Bucket to Flat Structure</h3>

  <p>Once you're confident that most users have updated to the transitional release from Step 1, you can switch your bucket/hosting to use the new flat structure:</p>

  <ol>
    <li>Build your app with Electrobun v1 - artifacts will now be generated with the flat naming convention</li>
    <li>Upload the new flat-named artifacts to your bucket</li>
    <li>Optionally remove the old folder-based artifacts</li>
  </ol>

  
```
# Using Electrobun v1
bun run build --env=canary
# Deploy the flat-named artifacts from artifacts/ folder
```


  <h3>Step 5: Release Updates</h3>

  <p>From now on, all releases will use the new flat structure. Users who updated in Step 1 will seamlessly transition to fetching updates from the new URLs.</p>

  <h2>Why This Change?</h2>

  <p>The folder-based structure worked well with S3 and R2 buckets, but GitHub Releases and many other hosting platforms don't support folder hierarchies. The new flat prefix-based naming works everywhere while maintaining clear organization through the prefix.</p>

  <h2>API Changes</h2>

  <p>If you're using the Updater API directly, note that `Updater.localInfo.bucketUrl()` has been renamed to `Updater.localInfo.baseUrl()`.</p>

  <h2>Need Help?</h2>

  <p>If you run into issues during migration, please open an issue on the <a href="https://github.com/nicksellen/electrobun">Electrobun GitHub repository</a>.</p>
