## Migration Notes

### End users will be logged out of any sites once after upgrading

              <p>This release bumps the internal CEF cache format version. On first launch after an end user receives an app update built with this Electrobun version, the CEF cache folder is wiped automatically. Users will be logged out of any sites they were logged into via your app and any client-side site data (cookies, localStorage, IndexedDB) is reset.</p>

              <p>The wipe is silent (no dialog), runs once per cache-format bump, and only affects the CEF cache folder. WebView2 storage and your own app data are untouched.</p>

              <p>This was necessary to recover cleanly from the CEF bump from 145 to 147, which previously surfaced "Profile error occurred" dialogs on launch when an existing CEF cache from an older version was reused.</p>

              <p>Future Electrobun releases only repeat this behavior when a similar incompatibility appears — routine CEF version bumps don't trigger another wipe.</p>

## New Features

              <ul>
                <li><strong>DirectComposition for transparent windows on Windows.</strong> Transparent windows now go through a DirectComposition compositor, bringing them to parity with the existing DComp path used by non-transparent windows. WGPU surfaces and WebView2 share a single window through a layered visual tree, including a zero-copy WGPU→DComp swap-chain bridge that brought the bridged path from ~48 FPS to ~1729 FPS in benchmarks. A native <code>WM_SIZE</code> resize hook keeps resizes flash- and jank-free, and a Windows 8.1+ feature gate falls back cleanly on older systems.</li>
                <li><strong>File associations.</strong> A new <code>app.fileAssociations</code> config declares document types your app handles. The CLI generates <code>CFBundleDocumentTypes</code> in <code>Info.plist</code> and copies document-type icons into the app bundle. On macOS, files opened via associations arrive as <code>file://</code> URLs through the existing <code>open-url</code> event — no new event type to wire up.</li>
                <li><strong>Native <code>.icon</code> support for macOS Liquid Glass.</strong> When <code>mac.icons</code> points to a <code>.icon</code> file from Icon Composer, the build pipeline runs <code>actool</code> to produce <code>Assets.car</code> for Liquid Glass on macOS 26+, with an <code>.icns</code> fallback for older versions. Existing <code>.iconset</code> → <code>iconutil</code> builds are unchanged. Both icon tools now check exit codes so broken icon builds fail fast instead of producing icon-less apps.</li>
                <li><strong><code>BrowserWindow.hide()</code>.</strong> First-class hide API on <code>BrowserWindow</code> with native bindings on all three platforms — no more offscreen-positioning hacks.</li>
                <li><strong>Linux desktop integration.</strong> Apps now drop a <code>.desktop</code> file into the XDG applications directory so they appear in the system app menu and launcher.</li>
                <li>Bumped Bun to 1.3.11 and CEF from 145 to 147, with improved handling of the new CEF permissions model.</li>
                <li>Display full error messages when <code>electrobun build</code> or <code>electrobun dev</code> fails, instead of swallowing the underlying output.</li>
                <li>Renamed <code>Updater.localInfo.getLocallocalInfo()</code> to <code>getLocalInfo()</code> (the documented name). The old name is kept as a deprecation alias.</li>
                <li>Exported the <code>RESOURCES_FOLDER</code> constant from <code>Electrobun.PATHS</code>.</li>
                <li>Moved Electrobun docs back into the main repo with a dedicated docs deploy workflow.</li>
              </ul>

## Bug Fixes

              <ul>
                <li>Fix <code>hiddenInset</code> frameless window on Windows — preserve resize borders and DWM shadow, and stop clipping content when maximized.</li>
                <li>Fix fullscreen behaviour and traffic light positioning on macOS.</li>
                <li>Use <code>xcrun</code> for macOS native builds so the right toolchain is picked up.</li>
                <li>Fix Windows tar handling and resolve <code>rcedit</code> explicitly from the dependency tree.</li>
                <li>Linux: persistent cookie storage now works under WebKitGTK; full-size BrowserViews render as a proper overlay instead of a 1×1 GtkFixed; CEF webviews stop opening in their own windows.</li>
                <li>Unify CEF partition path handling and use direct child paths for partitions.</li>
                <li>Strip trailing slashes from <code>views://</code> URL paths.</li>
                <li>Allow <code>null</code> for <code>BrowserView.ptr</code> to avoid crashes on torn-down views.</li>
                <li>Include template <code>.gitignore</code> files when scaffolding new projects.</li>
                <li>Remove the spinner-thread deadlock in the Linux extractor and improve its progress UX.</li>
                <li>Remove a duplicate <code>makeSurfaceTexture</code> definition in the WebGPU layer.</li>
              </ul>
